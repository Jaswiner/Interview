  • 🔗链接: Hashmap#put-源码分析

    # 存储方式
- Java8中使用位桶(数组)+链表/红黑树实现
- 链表长度超过阈值(8)时转成红黑树(减少查找时间)

# 底层原理与扩容
1.作为容器，HashMap主要是实现对容器内元素的增删改查功能，这些和她的底层数据结构分不开的

2.put()的时候，会先进行hash()计算

3.hash()计算先进行高位计算，将hashcode无符号右移16位后和hashcode进行异或位运算(^)后，本质上保留高位，将高位和地位进行了混合

4.hash()运算后生成值和数组的长度-1进行位与(&)运算，然后得到在数组中的位置

    # 线程安全吗
- 线程不安全

# 为什么
- 扩容的时候破坏了数组的结构，和ArrayList的本质相同

# 解决办法
- 使用Collections.synchronizedMap(new HashMap(...))
- 使用并发包中的ConcurrentHashMap
- 使用Hashtable
